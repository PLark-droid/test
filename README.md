## Getting Started

### Prerequisites

- Node.js 20+
- Lark API credentials (App ID & App Secret)
- GitHub token for Miyabi integration

```bash
# Set environment variables
cp .env.example .env
# Edit .env and add your tokens:
# - LARK_APP_ID
# - LARK_APP_SECRET
# - LARK_CALENDAR_ID (optional, defaults to 'primary')
# - GITHUB_TOKEN
```

### Installation

```bash
npm install
```

### Development

```bash
npm run dev          # Run Lark calendar app
npm run build        # Build project
npm test             # Run tests
npm run typecheck    # Check types
npm run lint         # Lint code
```

### Usage

```bash
# Run the app to get meeting suggestions
npm run dev

# Output will show:
# - Recommended meeting time slots
# - Daily available time slots
# - Duration of each slot
```

## Features

- **Lark Calendar Integration**: Connects to Lark (Feishu) Calendar API
- **Automatic Slot Detection**: Finds available time slots in your calendar
- **Working Hours Support**: Respects configured working hours (default: 9:00-18:00)
- **Smart Suggestions**: Recommends top meeting time slots
- **Customizable Duration**: Configure meeting duration (default: 60 minutes)
- **Multi-day Search**: Search for available slots across multiple days

## Miyabi Framework

This project uses **7 autonomous AI agents**:

1. **CoordinatorAgent** - Task planning & orchestration
2. **IssueAgent** - Automatic issue analysis & labeling
3. **CodeGenAgent** - AI-powered code generation
4. **ReviewAgent** - Code quality validation (80+ score)
5. **PRAgent** - Automatic PR creation
6. **DeploymentAgent** - CI/CD deployment automation
7. **TestAgent** - Test execution & coverage

### Workflow

1. **Create Issue**: Describe what you want to build
2. **Agents Work**: AI agents analyze, implement, test
3. **Review PR**: Check generated pull request
4. **Merge**: Automatic deployment

## Configuration

### Environment Variables

Required variables (see `.env.example`):

- `GITHUB_TOKEN` - GitHub personal access token
- `ANTHROPIC_API_KEY` - Claude API key (optional for local development)
- `REPOSITORY` - Format: `owner/repo`

### GitHub Actions

Workflows are pre-configured in `.github/workflows/`:

- CI/CD pipeline
- Automated testing
- Deployment automation
- Agent execution triggers

**Note**: Set repository secrets at:
`https://github.com/PLark-droid/test/settings/secrets/actions`

Required secrets:
- `GITHUB_TOKEN` (auto-provided by GitHub Actions)
- `ANTHROPIC_API_KEY` (add manually for agent execution)
